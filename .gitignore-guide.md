# 📋 Git Ignore Guide — What's NOT in the Repository

Este archivo documenta qué archivos son **locales a cada máquina** y nunca deben ser commiteados.

## ✅ Correctamente Excluidos (y por qué)

### 🔧 IDE & Editor Local Config
```
.idea/           → Android Studio workspace (personal settings, run configs)
.vscode/         → VS Code workspace settings (personal preferences)
*.iml            → IntelliJ IDEA project files
.rider/          → JetBrains Rider configuration
*.DotSettings*   → ReSharper / Rider user settings
```
**Por qué:** Cada desarrollador tiene preferencias distintas (temas, hotkeys, extensions).

### 📦 Build & Gradle Cache
```
ARES-mobile/.gradle/       → Gradle dependency cache (regenerado cada build)
ARES-mobile/build/         → Compiled outputs y intermediates
ARES-mobile/app/build/     → APK builds, compiled classes
.kotlin/                   → Kotlin compiler cache
```
**Por qué:** Se regeneran automáticamente y son MUY grandes (~500MB+).

### 🎯 Local Machine Config
```
ARES-mobile/local.properties   → SDK path, local secrets (NUNCA incluir!)
ARES-mobile/captures/          → Device screen captures
*.apk                          → Built APK files
*.aab                          → Android App Bundles
```
**Por qué:** Contiene rutas locales, versiones de SDK específicas, y binarios compilados.

### 🗂️ Generated & Runtime Data
```
ARES-desktop/AresAssistant/data/   → Config, chat history, downloaded models
ARES-desktop/AresAssistant/logs/   → Application logs y crash dumps
```
**Por qué:** Se generan al ejecutar la app, son específicos de cada usuario.

### 🛠️ Internal Tools
```
.superpowers/          → Copilot chat history
.claude/               → Claude session data
design-preview-server/ → Node server artifacts
```
**Por qué:** Metadata local de herramientas de desarrollo.

### 🍎 OS-Specific Files
```
.DS_Store              → macOS folder metadata
Thumbs.db              → Windows thumbnail cache
desktop.ini            → Windows shortcut info
*.swp, *~              → Editor temp files
```
**Por qué:** Generados por el SO, varían por máquina.

## 📌 Qué SÍ está en el Repo

### ✔️ Archivos que SIEMPRE deben estar tracked:
```
settings.gradle.kts            → Gradle configuration (compartido)
build.gradle.kts               → Build config (compartido)
gradle/wrapper/                → Gradle wrapper JAR (reproducible builds)
gradle.properties              → Gradle system properties (compartido)
gradlew / gradlew.bat          → Gradle wrapper scripts
.gitignore                      → Esta misma exclusión list
```

## 🚀 Para Nuevos Miembros del Equipo

Cuando clonas el repo:
1. Los archivos `.gradle/`, `.idea/`, etc. NO estarán presentes
2. La primera vez que abres Android Studio, se regeneran automáticamente
3. Tu `local.properties` debe tener tu propia ruta de SDK (Android Studio lo genera)
4. No edites estos archivos locales en commits — están ignorados por design

## ⚠️ Si Accidentalmente Haces Commit de Archivos Locales

```bash
# Remover del tracking sin borrar localmente:
git rm --cached -r .idea/
git rm --cached local.properties
git commit -m "chore: remove local IDE files from tracking"
```

El archivo ya tendrá `.gitignore` actualizado, así que futuras modificaciones no serán tracked.
